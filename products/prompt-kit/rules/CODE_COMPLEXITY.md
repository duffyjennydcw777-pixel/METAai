# 🧪 Code Complexity Levels (CCL)

> Система классификации кода для AI. Определяет объём тестирования.
> AI ОБЯЗАН классифицировать КАЖДОЕ изменение перед выполнением.

---

## Level 1 — 🟢 Trivial (Деплой без тестов)

**Примеры:**
- README, CHANGELOG, комментарии, документация
- CSS/стили (если не ломают layout)
- .gitignore, .editorconfig
- Логирование (добавление logger.info)
- Переименование переменных (без изменения логики)

**⚠️ НЕ Level 1 (хитрые случаи):**
- Конфиги, меняющие поведение сервиса (порты, URL, feature flags) → **Level 2**
- Переменные окружения в `.env` → **Level 2**

**Действие AI:** Выполнить → обновить CHANGELOG → готово.

---

## Level 2 — 🟡 Standard (3 теста)

**Примеры:**
- Новый endpoint / функция
- Баг-фикс (исправление логики)
- Рефакторинг (изменение структуры кода)
- Валидация входных данных
- Фильтрация / сортировка
- Новая модель данных

**Обязательные тесты:**
1. ✅ **Happy path** — базовый сценарий работает
2. ❌ **Error case** — ошибка обрабатывается корректно
3. 🔲 **Edge case** — граничные условия (пустой ввод, null, MAX_INT)

**Когда нужно больше 3 тестов:**
- >3 ветвлений (if/elif/else) → +1 тест на каждое
- Несколько типов данных → тест для каждого типа
- Внешний сервис → mock + timeout тест

**Действие AI:**
```
📊 Complexity: Level 2 — Standard
🧪 Testing: [3 теста: happy path, error, edge]
```

---

## Level 3 — 🔴 Complex (Тестить до 85% уверенности)

**Примеры:**
- Финансовые расчёты (биллинг, подписки, скидки)
- Аутентификация / авторизация
- Миграции базы данных
- Async/concurrent код
- Security-критичный код (crypto, tokens)
- Core-алгоритмы бизнес-логики
- Деплой на production

**Тестирование:**
1. Все тесты Level 2
2. + Integration tests (реальные зависимости)
3. + Regression tests (не сломали старое)
4. + Stress/edge (concurrent access, large data)
5. + Dry-run перед production
6. + Post-deploy мониторинг

**AI обязан:**
- Назвать процент уверенности
- Перечислить оставшиеся 15% рисков
- Предложить план отката

**Действие AI:**
```
📊 Complexity: Level 3 — Complex
🧪 Testing: [N тестов, 85% confidence]
⚠️ Remaining risks: [список]
🔙 Rollback plan: [план]
```

## 📌 Post-Mortem Protocol

Если баг проскочил мимо тестов:

1. **Код инцидента**: `CCL-XXX`
2. **Что случилось**: описание
3. **Какой уровень был назначен**: Level N
4. **Какой должен был быть**: Level N+1
5. **Действие**: Повысить уровень для этого типа задач НАВСЕГДА
6. **Запись**: добавить в anti-patterns

> **Правило:** при сомнении между уровнями — ВСЕГДА выбирай высший.
