## 🛡️ Security Audit

### Найденные уязвимости

| Severity | Category | Description | Location | Fix |
|----------|----------|-------------|----------|-----|
| 🟠 HIGH | Input Validation | Небезопасное использование `subprocess.run()` с `errors="replace"` | `agents/conductor.py`, `agents/auto_committer.py` | Использовать `errors="strict"`, добавить обработку ошибок кодировки |
| 🟡 MEDIUM | Path Traversal | Небезопасное добавление путей в `sys.path` | `agents/conductor.py:5-6` | Использовать `os.path.abspath()`, проверять корректность пути |
| 🟡 MEDIUM | Injection | Потенциальная уязвимость при работе с git-командами | `agents/auto_committer.py:git_*()` | Экранировать пути и аргументы, использовать `shlex.quote()` |

| 🟢 LOW | Configuration | Глобальные константы с потенциально опасными путями | `agents/config.py` | Использовать абсолютные пути, проверять права доступа |

### Рекомендации по харденингу
1. Добавить валидацию всех входных путей через `os.path.normpath()` и `os.path.abspath()`
2. Использовать `shlex.quote()` для экранирования аргументов командной строки
3. Реализовать белый список допустимых путей и команд
4. Добавить обработку исключений для всех сетевых и файловых операций
5. Использовать `logging` с уровнями безопасности
6. Проверять права доступа к файлам перед операциями
