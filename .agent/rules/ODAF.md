# 📐 ODAF — Стандарт промптинга для всех проектов

> **Обязателен для ВСЕХ проектов.** Загружается вместе с `GLOBAL.md` и `MODES.md`.
> ODAF = Outcome → Data → Action → Format.
> Применять при ЛЮБОМ аналитическом, стратегическом или диагностическом запросе.
> Источник: методология @artemiy.miller, адаптирована 2026-05-15.

---

## Что такое ODAF

Каждый серьёзный промпт (не тривиальный баг-фикс) должен быть структурирован по 4 блокам:

```
O — Outcome:  Что конкретно получим? (измеримый результат, не «анализ»)
D — Data:     Какие данные входят? (метрики, файлы, контекст)
A — Action:   Какие шаги выполняем? (методология)
F — Format:   В каком виде результат? (таблица, диаграмма, чек-лист)
```

---

## Когда использовать ODAF

| Ситуация | ODAF? | Пример |
|----------|:-----:|--------|
| Customer Journey анализ | ✅ ОБЯЗАТЕЛЬНО | Карта пути клиента ONYX |
| Ценовая стратегия | ✅ ОБЯЗАТЕЛЬНО | Сценарии A/B/C с break-even |
| Unit-экономика | ✅ ОБЯЗАТЕЛЬНО | CAC, LTV, churn, прогнозы |
| Стратегическое решение | ✅ ОБЯЗАТЕЛЬНО | Запускать 5-й продукт? |
| Архитектурный анализ | ✅ Рекомендуется | Выбор БД, рефакторинг |
| Баг-фикс | ❌ Избыточно | Используй Режим Детектив |
| Простой код | ❌ Избыточно | Добавь кнопку |
| Деплой | ❌ Избыточно | Используй Режим Запуск |

## Шаблон ODAF-блока

AI ОБЯЗАН начинать аналитический ответ с ODAF-блока:

```
O — Outcome: [Конкретный измеримый результат]
D — Data: [Список входных данных с источниками]
A — Action: [Методология / шаги анализа]
F — Format: [Формат вывода: таблица / график / чек-лист / roadmap]
```

### Пример (хороший):
```
O — Outcome: Карта пути клиента с точкой обрыва и 5 действиями для увеличения конверсии на 30%
D — Data: 15 подписчиков, 9 платящих, 7 free, 8 с нулевым трафиком, churn 0%, BUSINESS_METRICS.md
A — Action: 7-этапный Customer Journey + кривая эмоций + анализ drop-off + приоритизация
F — Format: Таблица 7 этапов, визуальная кривая эмоций, ТОП-5 действий с дедлайнами
```

### Пример (плохой):
```
O — Outcome: Проанализировать клиентов    ← слишком размыто
D — Data: Данные из базы                  ← какие данные? откуда?
A — Action: Посмотреть и сделать выводы   ← нет методологии
F — Format: Текст                         ← не структурировано
```

## Интеграция с MODES.md

ODAF усиливает следующие режимы:

| Режим | + ODAF = |
|-------|----------|
| 🔍 Разведка | Структурированное исследование рынка с метриками |
| ♟️ Шахматы | Стратегия с данными, а не интуицией |
| 💰 Торговец | Ценообразование на основе unit-экономики, а не «примерно» |
| 🏥 Клиника | Диагностика с KPI, а не «кажется тормозит» |

### Комбинация Режим + ODAF:
```
🎭 Режим: Торговец + ODAF

O — Outcome: Обоснованная ценовая стратегия с 3 сценариями и break-even timeline
D — Data: Текущие тарифы, конкуренты, CAC=0, 9 платящих, MRR 900₽
A — Action: Cost-plus + value-based + competitive benchmarking
F — Format: 3 сценария в таблице, анализ чувствительности, рекомендация
```

## ODAF для промптов агентов METAai

При создании промптов для AI-агентов (в `.agent/rules/`, в Telegram-ботах, в автоматизациях) — использовать ODAF как каркас:

### Структура промпта агента:
```markdown
## [Название задачи]

### ODAF
- **O**: Что агент должен выдать
- **D**: Какие данные агент получает на вход
- **A**: Какой алгоритм/логика обработки
- **F**: В каком формате ответ пользователю

### Контекст
[Бизнес-контекст, ограничения, правила]

### Примеры
[2-3 примера входа → выхода]
```

## 12 готовых ODAF-шаблонов (библиотека)

> Проверенные шаблоны из стратегической сессии 2026-05-15.
> Применимы к ЛЮБОМУ продукту портфеля.

### 🗺️ #1 Customer Journey Map
```
O — Полная карта пути клиента с точкой обрыва
D — Юзеры, подписки, трафик, churn, платежи, конкуренты
A — 7 этапов (осведомлённость → рассмотрение → решение → онбординг → вовлечённость → лояльность → отток)
F — Таблица этапов + кривая эмоций + ТОП-5 действий
```

### 💰 #2 Ценовая стратегия
```
O — Обоснованное решение по цене с 3 сценариями
D — Текущие цены, конкуренты, CAC, LTV, маржа, ROI клиента
A — Value-based + cost-plus + competitive + сценарный анализ
F — Бенчмарк-таблица, 3 сценария, анализ чувствительности, рекомендация
```

### 📊 #3 Unit-экономика
```
O — CAC, LTV, breakeven, 12-месячный прогноз, red flags
D — Платящие, ARPU, churn, расходы, concentration risk
A — Когортный анализ, сценарное моделирование (лучший/базовый/худший)
F — Таблица метрик, 12-месячный прогноз, анализ чувствительности
```

### 🎯 #4 Конкурентный анализ
```
O — Карта конкурентов с нашим позиционированием
D — 5-10 конкурентов, их цены, функции, слабости
A — Feature matrix + pricing comparison + SWOT
F — Сравнительная таблица, positioning map, наш moat
```

### 📝 #5 Value Proposition Canvas
```
O — Сильный оффер, который продаёт результат, а не функцию
D — Целевая аудитория, их боли, текущие альтернативы
A — Jobs-to-be-done + Pain/Gain analysis
F — Формула оффера: [Результат] + [Срок] + [Гарантия/Доказательство]
```

### 🔄 #6 Retention / Churn анализ
```
O — Причины оттока и 3 действия для удержания
D — Активность юзеров, payment history, support tickets
A — Когортный анализ, RFM-сегментация, early warning signals
F — Churn curve, сегменты риска, playbook удержания
```

### 🚀 #7 Go-to-Market план
```
O — Пошаговый план выхода на рынок с метриками
D — TAM/SAM/SOM, каналы, бюджет, timeline
A — Channel prioritization + launch sequence + metric targets
F — Канбан-план с метриками на каждом этапе
```

### 🧪 #8 A/B тест дизайн
```
O — Гипотеза + дизайн эксперимента + критерий успеха
D — Текущая метрика (baseline), целевое улучшение
A — Формулировка H0/H1, расчёт sample size, duration
F — Experiment card: гипотеза, метрика, длительность, решение
```

### 📈 #9 Growth Model
```
O — Модель роста с рычагами (levers) и ограничителями (ceilings)
D — Текущие метрики воронки, CAC, viral coefficient
A — AARRR framework + growth loops + bottleneck analysis
F — Воронка с числами, 3 рычага роста, ceiling analysis
```

### 🏗️ #10 Архитектурный аудит
```
O — Оценка здоровья кодовой базы с приоритизированным tech debt
D — LOC, dependencies, test coverage, deploy frequency
A — DORA metrics + dependency audit + complexity analysis
F — Health scorecard, tech debt backlog, refactoring roadmap
```

### 💼 #11 Investor Pitch Data
```
O — Ключевые метрики для питча / самоотчёта
D — MRR, growth rate, churn, LTV:CAC, runway
A — SaaS metrics framework + benchmark comparison
F — One-pager с 10 ключевыми метриками
```

### ⚡ #12 Sprint Planning
```
O — Приоритизированный спринт с estimated impact
D — Backlog, business goals, available hours
A — ICE scoring (Impact × Confidence × Ease)
F — Таблица задач с ICE score, sorted by priority
```

## Правила обновления

1. При обнаружении нового полезного шаблона → добавить в библиотеку (#13, #14...)
2. При провале шаблона (дал плохой результат) → обновить Action/Format
3. Каждый успешный ODAF-анализ → сохранять в `Second_Brain/06_Business/` для повторного использования
4. ODAF — живой документ. Совершенствуется с каждым проектом.
