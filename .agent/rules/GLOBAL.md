# 🌍 GLOBAL RULES — Обязательны для ВСЕХ проектов

> Эти правила загружаются AI-агентом ПЕРВЫМИ при каждом ответе.
> Нарушение = ПРОВАЛ.

---

## 🧠 Контекст и память

1. **Knowledge Items First**: В начале КАЖДОЙ беседы проверить KI. Особенно: `ai-anti-patterns`, `maker-profile-global`, `project-management-patterns`, `deployment-patterns`.
2. **session_state.md**: Каждые 15-20 сообщений обновлять в scratch-директории. Фиксировать: задачу, решения, данные, TODO. При потере контекста — перечитать.
3. **Anti-Pattern Registry**: ПЕРЕД деплоем, SSH, Python/datetime кодом — сверяться с `knowledge/ai-anti-patterns/artifacts/anti-patterns.md`.

---

## 📋 Документация

4. **CHANGELOG.md**: Проверять наличие в каждом проекте. Задача НЕ завершена пока CHANGELOG не обновлён.
5. **docs/ (Documentation 2.0)**: Если проект серьёзный — должна быть папка `docs/` с 4 столпами:
   - `1_PRODUCT.md` — что, для кого, фичи
   - `2_ARCHITECTURE.md` — стек, структура, ADR
   - `3_INFRASTRUCTURE.md` — серверы, порты, домены (БЕЗ паролей!)
   - `4_DEPLOY_RUNBOOK.md` — пошаговый деплой
6. **DECISIONS.md**: ПЕРЕД предложением архитектурного решения — проверить Decision Log. Не предлагать то, что уже отвергнуто.
7. **SOLUTION_PATTERNS.md**: ПЕРЕД написанием кода — проверить библиотеку паттернов. Если есть PAT-XXX — использовать.

---

## 🔒 Безопасность

6. **Секреты**: НИКОГДА не оставлять API-ключи, пароли, токены в коде или Markdown. ТОЛЬКО `.env`.
7. **chmod/chown**: После КАЖДОГО деплоя файлов на Linux — напоминать о правах.
8. **SSH**: ВСЕГДА `-p 2222`. ТОЛЬКО по IP, не по домену. Сверяться с таблицей серверов (SRV-001).
9. **sed через SSH из PowerShell**: ЗАПРЕЩЕНО. Создать Python-скрипт.

---

## 🏗️ Качество

10. **Zero Copy-Paste**: Если AI может создать файл, прописать конфиг, сгенерировать команду — делать самому. Не заставлять человека копировать.
11. **НЕ ЛОМАТЬ**: При обновлении кодовой базы — 100% уверенность, что ничего не сломано.
12. **Git-дисциплина**: После 5+ изменённых файлов — напоминать о коммите. Не уходить из сессии с незакоммиченными изменениями.
13. **Премиум, не MVP**: Создатель требует премиум-качество. Никаких минимальных поделок.
15. **Code Complexity Levels**: ПЕРЕД деплоем классифицировать изменение (Level 1/2/3). См. `CODE_COMPLEXITY.md`. Level 1 = без тестов, Level 2 = 3 теста, Level 3 = тестить до 85% уверенности.
16. **Multi-Agent Pipeline**: Для Level 2+ — запустить `python review.py review --level N`. Агенты проверят код автоматически.
