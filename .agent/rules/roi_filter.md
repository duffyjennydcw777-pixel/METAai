> AI ОБЯЗАН применять этот фильтр при обсуждении новых задач, фич и инвестиций времени.

## Классификация задач по ROI

Перед началом работы AI оценивает задачу:

| Тип | Знак | Описание | Примеры |
|-----|:----:|----------|---------|
| **∞ ROI** | ♾️ | Делается 1 раз, работает навсегда | Автоматизация, rules, паттерны, курсы |
| **Высокий ROI** | 📈 | Окупается за 1-2 использования | Баг-фикс, рефактор, CI/CD |
| **Средний ROI** | ➡️ | Окупается за месяц | Новая фича, документация |
| **Низкий ROI** | 📉 | Разовое действие, не масштабируется | Ручной отчёт, one-off скрипт |
| **Отрицательный** | 🔴 | Тратит время без возврата | Переделывание, мета-работа ради меты |

## Формат вывода (при обсуждении задач)

```
♾️ ROI: [∞/Высокий/Средний/Низкий]
⏱️ Вложение: [сколько времени]
🔄 Возврат: [что получаем и как часто]
📈 Compound: [растёт ли со временем]
```

## Правила приоритизации

1. **∞ ROI задачи = приоритет #1.** Автоматизация, паттерны, rules всегда важнее фич.
2. **Если задача = Низкий ROI → предложить ∞ альтернативу.** «Вместо ручного отчёта — давай автоматизируем.»
3. **Мета-работа ≤ 10% времени.** Если ROI фильтр сам становится тратой времени — стоп.
4. **Compound > One-shot.** Задача, которая растёт со временем, всегда лучше разовой.
